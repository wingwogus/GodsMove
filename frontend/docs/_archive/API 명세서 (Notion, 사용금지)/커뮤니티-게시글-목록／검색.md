# 커뮤니티 게시글 목록/검색

> ⬆ 상위: [API 명세서](README.md)

### Response Body
```json
{
  "success": true,
  "data": {
    "items": [],
    "nextCursor": null
  },
  "error": null
}
```
### DTO
- Request: CommunityPostSearchRequest
- Response: CommunityPostListResponse
### Query Example
```javascript
GET /api/v1/community/posts?cropId=00000000-0000-0000-0000-000000000000&postType=QUESTION&keyword=발아&likedOnly=false&mineOnly=false&sort=POPULAR&cursor=<opaque-cursor>&size=20
```
### Query
- `cropId`: uuid, optional. 선택한 작물 게시판만 조회한다.
- `postType`: enum, optional. `GENERAL`, `QUESTION`.
- `keyword`: string, optional. 제목+본문 검색.
- `likedOnly`: boolean, optional. 기본값 `false`. true면 내가 좋아요한 게시글만 조회.
- `mineOnly`: boolean, optional. 기본값 `false`. true면 내가 작성한 게시글만 조회.
- `sort`: enum, optional. 기본값 `LATEST`. `LATEST`, `LIKE`, `COMMENT`, `POPULAR`.
- `cursor`: string, optional. 이전 응답의 `nextCursor`를 그대로 전달한다.
- `size`: number, optional. 기본값 `20`. 1 이상의 값이어야 한다.
### Rule
- opaque cursor 페이지네이션을 사용한다. 클라이언트는 cursor를 파싱하거나 직접 생성하지 않는다.
- `nextCursor`가 `null`이면 다음 페이지가 없다. 별도 `hasNext` 필드는 사용하지 않는다.
- 삭제된 게시글은 조회하지 않는다.
- `POPULAR` 점수는 `likeCount + commentCount`이며, 동점이면 최신순으로 정렬한다.
