# 온보딩 완료

> ⬆ 상위: [API 명세서](README.md)

### Rule
온보딩 마지막 단계에서 member 필수 프로필, 농장 정보, 선택 작물을 한 번에 저장한다. 요청에는 Authorization bearer token이 필요하며, 백엔드는 token의 member id를 사용한다.
성공 시 토큰은 재발급하지 않고 `OnboardingCompleteResponse`를 반환한다. 모든 member 필수 필드가 저장되면 `onboarding.status = COMPLETE`, `missingFields = []`가 된다.
### Request 핵심 변경
- `region`은 더 이상 받지 않는다.
- `profileMediaId`는 선택값이다. 이미지 업로드 API로 만든 `PROFILE` 용도 미디어 ID만 허용한다.
- `experienceLevel`은 0~100 숫자다.
- `managementType`은 `AGRICULTURAL_MANAGER_INDIVIDUAL`, `AGRICULTURAL_MANAGER_CORPORATION`, `FARMER_NON_MANAGER` 중 하나다.
- `farm` 객체를 필수로 받으며, 최소 `name`, `roadAddress`, `latitude`, `longitude`가 필요하다.
- `cropIds`는 하나 이상 필수이며, 온보딩 완료 시 `member_crops`가 생성된다.
### Response 핵심 변경
- 응답에 생성된 `farm`과 선택된 `crops`를 함께 내려준다.
- `farm`은 도로명/지번 주소, 대표 좌표, PNU, 지목, 면적, 경계 좌표, 데이터 출처를 포함한다.
