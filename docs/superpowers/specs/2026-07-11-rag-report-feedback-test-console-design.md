# RAG 기록 피드백·리포트 테스트 콘솔 설계

## 목적

`frontend/dev-rag-test.html` 하나로 로컬 백엔드의 기록 피드백 자동 생성과
주기 리포트 통계를 점검한다. 제품 화면을 흉내 내는 것이 아니라 개발자가
요청·상태 전이·공개 응답을 빠르게 검증하는 운영형 테스트 도구다.

## 시각 방향

- 시각 논지: 흰색 작업면, 짙은 잉크색 텍스트, 초록색 단일 강조색을 사용하는
  조용하고 밀도 높은 농업 운영 콘솔
- 콘텐츠 구성: 연결·인증 설정, 기록 피드백 작업면, 리포트 통계 작업면,
  원본 응답 인스펙터
- 상호작용 논지: 탭 전환, 폴링 중 상태 점멸, 응답 도착 시 결과 영역의 짧은
  reveal만 사용한다.

## 기능 범위

### 연결과 인증

- 기본 URL은 `http://127.0.0.1:8080`이다.
- 이메일 로그인 또는 수동 JWT 입력을 지원한다.
- URL과 자주 쓰는 ID는 `localStorage`, JWT는 `sessionStorage`에 저장한다.
- HTML은 로컬 CORS 허용 origin인 `http://127.0.0.1:5173`에서 제공한다.

### 기록 피드백

- JSON 요청 본문으로 영농기록을 생성하거나 수정한다.
- 생성 성공 시 반환된 `recordId`를 자동 반영하고 피드백 폴링을 시작한다.
- `GET /api/v1/farming-records/{recordId}/coaching-feedback`으로 상태를 조회한다.
- `PENDING`, `READY`, `FAILED`, `STALE` 상태와 `inputPrepared`를 표시한다.
- READY에서는 공개 계약인 `goodPoint.text`, `nextActions[].text/due/category`만
  사용자 결과 영역에 렌더링한다.
- FAILED에서만 재생성 버튼을 활성화한다.
- 상태 미리보기 모드는 백엔드를 호출하지 않고 네 상태의 UI만 검증한다.

### 주기 리포트

- 농장 ID와 작물 ID로 현재/직전 주기를 조회한다.
- 완료 리포트 목록을 조회하고 선택한 리포트의 상세를 조회한다.
- 현재 또는 직전 스냅샷을 선택하고 작업 유형 탭을 전환한다.
- 각 작업 객체에 직접 포함된 공통 통계와 유형별 분포·합계를 함께 표시한다.
- 리포트 코칭은 아직 구현되지 않았으므로 콘솔에 가짜 코칭 결과를 만들지 않는다.

### 원본 응답

- 모든 API 응답과 오류를 우측 인스펙터에 JSON으로 표시한다.
- 토큰 키가 포함된 값은 마스킹한다.
- HTTP 상태와 호출 경로를 요청 로그에 누적한다.

## 오류 처리

- 네트워크/CORS 실패는 base URL과 `5173` origin 확인 문구를 보여준다.
- JSON 요청 본문 파싱 실패는 호출 전에 차단한다.
- API 오류 응답은 상태 코드와 서버 응답을 그대로 인스펙터에 남긴다.
- 폴링은 READY/FAILED/STALE에서 자동 종료하며 최대 60회로 제한한다.

## 검증 기준

- 데스크톱 1440×900과 모바일 390×844에서 겹침이나 가로 스크롤이 없다.
- 탭, 미리보기 상태, JSON 파싱 오류, 기록 ID 자동 반영이 동작한다.
- READY 공개 결과 외 내부 근거·모델·감사·스냅샷은 사용자 결과 영역에 나오지 않는다.
- 외부 라이브러리나 새 빌드 의존성을 추가하지 않는다.
