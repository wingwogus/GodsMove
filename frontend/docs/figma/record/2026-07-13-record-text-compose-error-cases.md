# 텍스트로 기록하기 — 입력 검증(에러) 케이스

- 기록일: 2026-07-13 (사용자 제공 참고 이미지 3장 기반 — Figma 노드 캡처 아님, 규칙/문구만 전사)
- 대상: 텍스트 기록 작성 폼의 필수/제한 검증 표시. "완료" 탭 시(또는 blur 시) 미충족 필드에
  에러 표시.

## 에러 표시 규칙 (핵심 변형 로직)

폼은 **한 영역(section)에 필드가 2개**인 경우가 있다(진행 작물 = 농지+작물, 작업 내용 =
진행 작업 드롭다운 + 작업 내용 textarea). 이때:

1. **에러 상태 필드에만 개별로 red border**(빨강 테두리)를 준다. 유효한 필드는 기본 테두리 유지.
2. **에러 문구는 영역당 1줄**(그 영역 하단 헬퍼 자리)로, **무효한 필드 이름을 열거**해 조합한다.
   - 둘 다 무효 → "A와 B는 필수로 …해주세요."
   - 하나만 무효 → "A는 필수로 …해주세요." (그 필드만)
   - 동사는 필드 성격에 따라: 선택형(드롭다운)=**선택**, 작성형(textarea)=**작성**.
3. 문자수 초과 등 별도 규칙 위반은 전용 문구로 대체(아래).

즉 variant는 **필드별 상태(정상/에러) × 영역별 조합 문구**로 구성된다. 필드 컴포넌트는
`errorMessage` 유무로 red border 파생(기존 `AppFieldContainer`/`AppTextField`/`AppDropdown`/
`AppTextEditor`의 error 상태와 동일), 영역 문구는 화면 로직이 조합.

---

## 1. 진행 작물 영역 (농지 + 작물)

동사 = **선택** (둘 다 드롭다운).

| 농지 | 작물 | red border | 영역 에러 문구 |
|---|---|---|---|
| 미선택 | 미선택 | 농지 + 작물 | **농지와 작물은 필수로 선택해주세요.** |
| 미선택 | 선택 완료 | 농지만 | **농지는 필수로 선택해주세요.** |
| 선택 완료 | 미선택 | 작물만 | **작물은 필수로 선택해주세요.** |
| 선택 완료 | 선택 완료 | 없음 | (에러 없음) |

## 2. 작업 내용 영역 (진행 작업 드롭다운 + 작업 내용 textarea)

동사 = **작성** (영역 문구 기준). 필수 검증:

| 진행 작업 | 작업 내용 | red border | 영역 에러 문구 |
|---|---|---|---|
| 미선택 | 미작성 | 드롭다운 + textarea | **진행 작업과 내용은 필수로 작성해주세요.** |
| 선택 완료 | 미작성 | textarea만 | **작업 내용은 필수로 작성해주세요.** |
| 미선택 | 작성 완료 | 드롭다운만 | **진행 작업은 필수로 작성해주세요.** |
| 선택 완료 | 작성 완료 | 없음 | (에러 없음) |

### 문자수 초과 (작업 내용 textarea, 최대 500자)

| 상태 | 카운터 | red border | 문구 |
|---|---|---|---|
| 500자 초과 입력 | **500/500** (빨강) | textarea | **작업 내용은 최대 500자까지 작성 가능합니다.** |

- 초과 시 카운터 `n/n`도 빨강으로 표시. (정상 시 `#878787`)
- 초과 문구는 필수(미작성) 문구보다 우선(작성은 됐으나 길이 위반).

## 확정 에러 문구 목록 (워딩 그대로)

진행 작물:
- `농지와 작물은 필수로 선택해주세요.`
- `농지는 필수로 선택해주세요.`
- `작물은 필수로 선택해주세요.`

작업 내용:
- `진행 작업과 내용은 필수로 작성해주세요.`
- `작업 내용은 필수로 작성해주세요.`
- `진행 작업은 필수로 작성해주세요.`
- `작업 내용은 최대 500자까지 작성 가능합니다.`

## 구현 메모 (참고)

- 진행 작물/작업 내용처럼 **2필드 1영역**은 필드 각각 `errorMessage`를 두지 말고, **영역
  단위 검증 결과**로 (a) 각 필드의 error(red border) on/off, (b) 영역 하단 조합 문구 1줄을
  파생하는 편이 이미지와 일치. 기존 `AppFieldContainer`의 helper/error 라인은 필드마다
  달리므로, 2필드 영역은 필드의 헬퍼를 숨기고 **영역 전용 에러 라인**을 별도로 두는 방식 검토.
- 단일 필드 영역(날짜, workType별 상세 필드 등)은 필드 자체 `errorMessage`로 충분.
- 이 문서는 워딩/조합 규칙 기록용. 다른 영역(날짜·상세 필드)의 개별 필수 문구는 미제공 —
  필요 시 동일 패턴("X는 필수로 선택/작성해주세요.")으로 확장 추정하되 확정 전 확인.
- 트리거 시점(완료 탭 시 일괄 vs blur 시 개별)은 이미지에서 불명 — 구현 전 확인.

## 구현 반영 (2026-07-13)

- **완료 버튼은 "disabled 표시(회색)이지만 탭 가능"**: `AppButton(appearsDisabled:)` 신규 옵션으로
  필수 미충족 시 회색으로 보이되 눌리게 함(`.disabled()`는 탭까지 막으므로 미사용). 탭 시
  `showValidation=true`로 전환되어 에러가 표시되고, 유효할 때만 실제 제출.
- **트리거 = 완료 탭 시 일괄**. 탭 전에는 에러 미표시(입력 중 방해 없음).
- 현재 에러 표시 범위: 진행 작물(농지·작물 조합 문구), 작업유형, 메모(필수/30자↑/500자↓),
  그리고 workType 상세 필수 필드(심은 방법·씨앗량/모종수, 사용 비료·비료 사용량, 사용 농약·
  농약 사용량·총 살포량). 물주기/잡초 진행 방식은 API optional이라 제출 비차단(UI `*` 표기와의
  차이는 [충돌 C-5/C-13](2026-07-13-record-backend-conflicts.md)).
- 2필드 1영역 조합 문구는 진행 작물에만 완전 적용. 작업/메모·상세 필드는 필드별 문구(근사) —
  완전 조합 문구는 후속 정교화.
