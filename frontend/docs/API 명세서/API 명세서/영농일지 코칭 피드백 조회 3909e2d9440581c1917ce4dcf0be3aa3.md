# 영농일지 코칭 피드백 조회

API Path: /api/v1/farming-records/{recordId}/feedback
Index: RECORD-008
Method: GET
Request: None
Response: CoachingFeedbackResponse

## Response Fields

- `feedbackId`: uuid
- `recordId`: uuid
- `status`: `PENDING | READY | FAILED | STALE`
- `sourceRevision`: number
- `inputPrepared`: boolean. 실제 엔티티 기반 입력 스냅샷 준비 여부
- `failureCode`: string, nullable
- `feedback`: object, nullable. `READY` 상태에서만 제공
  - `goodPoint.text`: string
  - `nextActions[]`
    - `text`: string
    - `due`: `TODAY | THIS_WEEK | NEXT_WEEK | NEXT_CHECK`
    - `category`: `WEATHER | PEST_DISEASE | IRRIGATION | FERTILIZING | PEST_CONTROL | HARVEST | CULTIVATION | GENERAL`
- `createdAt`, `updatedAt`: datetime

## Rule

인증 회원이 소유하고 삭제되지 않은 영농일지만 조회한다. 현재 원본 revision의
피드백을 우선 반환하고, 현재 revision 행이 없을 때만 가장 최신 `STALE` 행을
반환한다.

영농일지를 저장하면 서버가 자동으로 현재 revision의 피드백 행을 `PENDING`으로
만들고, 커밋 후 엔티티 기반 입력 스냅샷을 준비한 뒤 RAG/LLM 생성을 실행한다.
생성 성공 시 `READY`, 생성 실패 시 `FAILED`가 된다. 이 GET API는 생성 트리거가
아니라 현재 상태를 폴링/조회하는 API다.

`feedback`은 `READY` 전까지 `null`이다. `READY`에서도 공개 필드는
`goodPoint.text`와 `nextActions[].text/due/category`뿐이다. 입력 스냅샷,
근거/basis, evidence refs, citations, 모델명, 감사 상태/경고는 서버 내부 보관
대상이며 응답에 노출하지 않는다.

수동 재생성은 자동 생성이 `FAILED`로 끝난 경우에만 허용한다. `PENDING`,
`READY`, `STALE` 상태의 중복 조회나 중복 생성 요청은 같은 revision에 추가
피드백 행을 만들지 않는다.
