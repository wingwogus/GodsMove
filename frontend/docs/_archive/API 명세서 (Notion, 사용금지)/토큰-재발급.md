# 토큰 재발급

> ⬆ 상위: [API 명세서](README.md)

### Rule
refresh token으로 accessToken과 refreshToken을 재발급한다. 서버는 request body의 `refreshToken`을 먼저 사용하고, 없으면 `refreshToken` cookie를 사용한다. 둘 다 없으면 인증 실패로 처리한다.
### Request
`ReissueRequest` 또는 `refreshToken` cookie
### Response
`TokenResponse`
응답 헤더에 새 `refreshToken` cookie를 `Set-Cookie`로 내려준다.
