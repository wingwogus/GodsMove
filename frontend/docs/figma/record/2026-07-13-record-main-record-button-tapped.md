## 현재 구현과의 차이 (후속 반영 항목)

1. **FAB inert → 스피드다이얼**: 현재 `RecordListView`의 FAB는 무동작. 이 캡처대로
   탭 시 딤 + 음성/텍스트/닫기 스피드다이얼을 열도록 교체 필요. (모드 선택까지가 이 화면
   범위, 실제 작성 화면 진입은 미캡처.)
2. **활성 필터 칩 스타일/라벨**: 현재는 선택 이름만 표기(비활성 스타일 유지). 디자인은
   활성 시 초록 아웃라인 + 카운트(`작물 4`) 표기 → 반영 필요. 단, 카운트는 다중 선택
   전제이므로 API 다중 선택 협의와 함께 처리.
3. **딤값**: 공통 `#1a1a1a`@64% 스크림을 스피드다이얼/바텀시트 공통으로 적용.
