# 리포트 피드백 출력 허용 범위 완화 설계

## 목적

리포트 코칭의 최대 65자 및 근거 검증 계약은 유지하면서, 모델이 만든 개행과
역할별 최소 길이 차이 때문에 정상적인 내용까지 실패하는 비율을 낮춘다.

## 출력 계약

- `summary`와 모든 섹션 `text`는 20~65자로 통일한다.
- `summary`와 모든 섹션 `text`의 `CRLF`, `CR`, `LF` 개행은 검증 전에 단일
  공백으로 치환한다.
- 개행 앞뒤의 공백도 함께 정리해 결과에 불필요한 연속 공백을 남기지 않는다.
- 프롬프트에는 여전히 한 문단 출력을 요청하되, 모델이 개행을 반환해도
  `*_text_paragraph` 오류로 실패시키지 않는다.

## 유지하는 검증

- 최대 65자
- 필수 섹션별 정확한 항목 개수
- 빈 `basis`와 빈 `text` 거부
- `evidenceRefs` 필수 및 허용 목록 검증
- 비교 항목의 현재·이전 리포트 근거 필수
- 구조화 JSON 파싱과 재시도 흐름

기록 피드백의 길이·문단 정책은 변경하지 않는다. API와 데이터베이스 스키마,
기존 저장 결과도 변경하지 않는다.

## 처리 흐름

OpenClaw 응답을 `ReportFeedbackContent`로 변환한 직후 공개 문구의 개행을
정규화한다. 정규화된 결과를 기존 검증기에 전달하고, 검증을 통과한 동일 객체를
저장한다. 따라서 저장 데이터와 검증 대상이 일치한다.

## 검증

- 개행이 포함된 유효 응답이 재시도 없이 성공하고 저장 결과에는 공백만 남는다.
- 모든 공개 문구가 정확히 20자일 때 허용된다.
- 모든 공개 문구가 19자일 때 기존 길이 진단으로 거부된다.
- 65자는 허용하고 66자는 거부한다.
- 프롬프트가 모든 공개 문구에 20~65자 범위를 안내한다.
