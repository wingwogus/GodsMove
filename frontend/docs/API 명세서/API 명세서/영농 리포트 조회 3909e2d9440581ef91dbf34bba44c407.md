# 영농 리포트 조회

API Path: /api/v1/farming-reports
Index: REPORT-001
Method: GET
Response: FarmingReportResponse

## 1단계 Endpoints

- `GET /api/v1/farming-reports/current?farmId={uuid}&cropId={uuid}`
- `GET /api/v1/farming-reports?farmId={uuid}&cropId={uuid}&cursor={opaque}&size=20`
- `GET /api/v1/farming-reports/{reportId}`
- `GET /api/v1/farming-reports/{reportId}/feedback`

## Current

회원·밭·작물 조합의 ACTIVE 리포트와 직전 COMPLETED 리포트를 함께 반환한다.

- ACTIVE 리포트가 없으면 404가 아니라 `current: null`을 반환한다.
- `previous`는 지난해 리포트가 아니라 직전 완료 주기다.
- 현재 주기가 마지막 수확으로 닫힌 직후에는 `current: null`, `previous: COMPLETED`가 된다.

## List

완료된 주기 목록만 반환한다.

- 목록 항목은 `COMPLETED` metadata만 포함한다.
- `ACTIVE`, `SUPERSEDED`, `statistics`는 목록 응답에 포함하지 않는다.
- 정렬 기준은 완료 경계인 마지막 수확 시각과 마지막 수확 기록 ID다.

## Detail

선택한 리포트의 전체 통계와 비교 대상 리포트를 함께 반환한다.

- `selected`는 요청한 리포트의 전체 detail이다.
- `previous`는 `selected` 바로 이전 COMPLETED 주기의 전체 detail이며 없으면 `null`이다.
- 화면은 `selected` 하나로 전체 화면과 작업별 화면을 함께 구성한다.

## Feedback

완료 리포트가 저장되면 서버가 자동으로 피드백을 생성한다. 이 endpoint는
`PENDING`, `READY`, `FAILED` 상태를 반환하므로 클라이언트는 PENDING 동안 조회를
재시도한다. READY 결과는 `summary`, `strengths`, `improvements`,
`nextCycleActions`만 노출하며 생성 근거·감사 데이터는 응답에 포함하지 않는다.

## Rule

리포트는 회원·밭·작물 조합별 영농 주기 단위로 저장된다. 앞선 마지막 수확이 없으면 최초 기록부터 시작하고, 앞선 마지막 수확이 있으면 그 다음 기록부터 시작한다. 일부 수확은 주기를 닫지 않으며 마지막 수확만 해당 기록을 포함해 주기를 닫는다. 심기는 강제 시작점이 아니며 심기 전 준비 작업도 같은 주기에 포함될 수 있다. 관수 횟수와 간격만으로 수분 부족·과다·적정을 판단하지 않는다.
