# 카카오 로그인

> ⬆ 상위: [API 명세서](README.md)

### Rule
앱에서 받은 Kakao OIDC ID token과 raw nonce를 검증한다. `kakaoAccessToken`이 있으면 Kakao OIDC userinfo를 조회해 email, name, phone, birthDate prefill에 사용한다.
검증 성공 시 `ExternalIdentity(provider=KAKAO, providerSubject=sub)`로 member를 찾거나 생성하고 `LoginResponse`를 반환한다.
### 처리 기준
- ID token의 issuer, audience, nonce, replay를 검증한다.
- userinfo를 조회한 경우 userinfo `sub`가 ID token subject와 같아야 한다.
- 기존 `external_identity`가 있으면 email을 다시 요구하지 않는다.
- 신규 identity는 email 없이도 생성할 수 있다.
- provider에서 받은 profile 값은 member의 빈 필드에만 prefill한다.
- 온보딩 상태는 현재 member 필수 필드 기준으로 계산한다.
