# 토큰 재발급

API Path: /api/v1/auth/reissue
Index: AUTH-006
Method: POST
Request: ReissueRequest (ReissueRequest (../DTO(%EB%8D%B0%EC%9D%B4%ED%84%B0%20%EC%A0%84%EB%8B%AC%20%EA%B0%9D%EC%B2%B4)/ReissueRequest%203929e2d9440581c0a4d6dd81e6762b39.md)) or refreshToken cookie
Response: TokenResponse (TokenResponse (../DTO(%EB%8D%B0%EC%9D%B4%ED%84%B0%20%EC%A0%84%EB%8B%AC%20%EA%B0%9D%EC%B2%B4)/TokenResponse%203909e2d9440581139390e591613d6fcb.md))

## Rule

refresh token으로 accessToken과 refreshToken을 재발급한다. 서버는 request body의 `refreshToken`을 먼저 사용하고, 없으면 `refreshToken` cookie를 사용한다. 둘 다 없으면 인증 실패로 처리한다.

## Request

`ReissueRequest` 또는 `refreshToken` cookie

## Response

`TokenResponse`

응답 헤더에 새 `refreshToken` cookie를 `Set-Cookie`로 내려준다.
