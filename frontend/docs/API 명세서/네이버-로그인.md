# 네이버 로그인

> ⬆ 상위: [API 명세서](README.md)

### Rule
앱에서 받은 Naver access token으로 Naver profile API를 호출한다. `response.id`를 providerSubject로 사용한다.
검증 성공 시 `ExternalIdentity(provider=NAVER, providerSubject=response.id)`로 member를 찾거나 생성하고 `LoginResponse`를 반환한다.
### 처리 기준
- Naver profile API 응답의 id를 providerSubject로 저장한다.
- email은 있으면 member email 후보로 사용한다. 이미 다른 member가 같은 email을 사용 중이면 저장하지 않을 수 있다.
- name, mobile, birthyear + birthday는 member의 빈 필드에만 prefill한다.
- 기존 `external_identity`가 있으면 email을 다시 요구하지 않는다.
- 신규 identity는 email 없이도 생성할 수 있다.
- 온보딩 상태는 현재 member 필수 필드 기준으로 계산한다.
