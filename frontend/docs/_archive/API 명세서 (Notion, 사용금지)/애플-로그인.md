# 애플 로그인

> ⬆ 상위: [API 명세서](README.md)

### Rule
Apple identity token을 Apple JWKS로 검증한다. raw nonce를 SHA-256 hex로 변환해 token nonce claim과 비교한다.
검증 성공 시 `ExternalIdentity(provider=APPLE, providerSubject=sub)`로 member를 찾거나 생성하고 `LoginResponse`를 반환한다.
### 처리 기준
- Apple identity token의 issuer, audience, nonce, replay를 검증한다.
- `userIdentifier`가 전달되면 token subject와 같은지 확인한다.
- 기존 `external_identity`가 있으면 email을 다시 요구하지 않는다.
- Apple은 최초 로그인 이후 email을 다시 주지 않을 수 있다.
- 신규 identity는 email 없이도 생성할 수 있다.
- MVP에서는 Apple authorization code 저장/교환과 Apple name prefill을 처리하지 않는다.
- 온보딩 상태는 현재 member 필수 필드 기준으로 계산한다.
